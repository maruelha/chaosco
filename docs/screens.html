Core
Dashboard GET /

The home screen. Shows a grid of action cards — one per major module. Acts as the main navigation hub since there is no persistent sidebar.

Cards Inbox (pending count badge) · Import · Defects · Core South Spillover · Retail · Meeting Prep · To-Do · Follow-ups · Links · Contacts · CS Follow-Up Tracker · Encouragements (undelivered count badge) · Enhancements (open count badge) Primary action Run Import — triggers a POST to /import and redirects to the Import Result screen Navigation Each card contains a link button to the corresponding list screen. The site title in the header also links back here from every page.
Inbox GET /inbox

Daily capture pad. Paste notes and screenshots during the day, then file each item to its destination (Defect, Retail test case, Spillover, Test Learning, or Follow-up) when ready. Unfiled items stay here until moved or deleted. Dashboard card shows pending count badge.

Add item Heading (optional) + large textarea + 📷 Add screenshot or Ctrl+V paste. Items appear in reverse-chronological order. Per-item actions Edit — inline toggle; updates heading and note text
Delete — removes item and all attached screenshots
File › — opens filing picker: choose type, search by name, select target, then Move › Filing behaviour Single SQL UPDATE — the note row and all its attachments move to the target entity. Filed items appear immediately in the target's detail page notes log. Screenshot support PNG/JPG/GIF/WEBP. Ctrl+V pastes a Snipping Tool image into the last-hovered item. Thumbnails shown inline; ✕ to delete. Table source notes (entity_type='input', entity_id='inbox') + attachments
Import Result POST /import

Shown after running an import. Displays a per-tab summary of what happened — how many rows were inserted, updated, or skipped, plus archive status and any skip-log file path.

Per-tab summary Defects · Core South Spillover · Retail (only tabs with enabled: true in settings.yaml appear) Counts shown Inserted · Updated · Skipped (blank key) · Skipped (duplicate) · Ignored (blank rows) Archive info Archive status (archived / skipped_duplicate / disabled) and path of archived file Skip log Path to the per-run CSV if any rows were skipped Navigation Back to Dashboard
Defects
Defects — List GET /defects

Paginated list of all imported defects with inline filters. The default view hides statuses "Withdrawn" and "Confirmed" (configurable in settings.yaml).

Columns Defect ID · SolMan Name · Blocked TCs · Channel · Status · Priority · Assigned To · Date Reported · Prod · DTC O2C (inline checkbox) · Daily (inline checkbox) Blocked TCs column Count of Retail test cases whose defect_id_ref references this defect. Computed as a subquery (LIKE match — not stored). Shows — when zero. When non-zero, displayed as a red badge link that opens the Retail list pre-filtered by that defect ID (?search_defect=<id>). Filters Search (defect ID) Channel Status (multi-select) Action needed DTC O2C Daily Show all toggle Row actions Click row → Defect Detail · Add note shortcut per row · Blocked TCs badge → Retail list filtered by defect ID Flash banners Green "Note added." banner when returning from a successful note add Table source defects LEFT JOIN defect_annotations + subquery on retail.defect_id_ref
Defects — Detail GET /defects/<defect_id>

Full view of a single defect — imported fields on the left, editable annotation form on the right, and a notes log at the bottom.

Read-only fields All columns from the defects table (channel, status, priority, dates, scenario, etc.) Editable annotation Description · Business Impact · Reach · Retest Needs · Next Step · Action Needed (checkbox) · Comments · DTC O2C (checkbox) · DTC O2C Responsible (text) · To discuss on daily (checkbox)
Save → POST to same URL → redirect with ?saved=1 flash
DTC O2C and Daily can also be toggled inline from the list view (AJAX, no reload) Notes section List of all notes for this defect (heading, body, timestamp). Per-note Edit and Delete links. Each note also shows image thumbnail gallery with a 📷 Add screenshot button. Hover a thumbnail to reveal the ✕ remove button. Ctrl+V pastes a clipboard image (e.g. from Snipping Tool) into the last-hovered note. Add to Meeting Prep 📅 Add to Meeting Prep button at the bottom — expands an inline form: pick meeting + edit pre-filled topic (defect ID + SolMan name). POSTs to /meeting-prep/add with source_entity_type=defect and source_entity_id. Redirects back with green confirmation banner. Flash banners Annotation saved · Note added · Note saved · Note deleted · Added to Meeting Prep Navigation ← Back to Defects · Add note · 📅 Add to Meeting Prep Table source defects LEFT JOIN defect_annotations + notes + attachments
Defects — Note Add / Edit / Delete GET /defects/<id>/notes/add  ·  /notes/<id>/edit  ·  /notes/<id>/delete

Three separate screens for managing notes on a defect. Add and Edit share the same note_form.html template (distinguished by a mode variable). Delete is a confirmation page. Screenshot attachments are managed via AJAX on the detail page itself (no separate screen).

Add / Edit fields Heading (optional) · Note text (required — shows inline error if empty) Delete Confirmation page showing the note heading/body. Requires a second POST to confirm deletion. Return flow Add can return to either the defect detail or the defects list (controlled by return_to query param). Edit and Delete always return to the defect detail. Screenshot attachments Upload via file picker or Ctrl+V paste from Snipping Tool (AJAX, no page reload). Delete via ✕ button on thumbnail hover. Files stored in data/uploads/. Routes: POST /notes/<note_id>/attachments/add · POST /notes/<note_id>/attachments/<id>/delete · GET /uploads/<filename>. Tables written notes (entity_type = 'defect') · attachments (on upload/delete)
Core South Spillover
Spillover — List GET /spillover

Scrollable table of Core South Spillover items. Rows are kept compact — only Excel-imported fields plus Next Step are shown as columns. Long annotation fields (Report Group, Comment for Sign-Off, Importance) are moved into a per-row Details popup. Four action buttons per row open popup dialogs or navigate to the detail page.

Columns Excel Row · Area · Status · Name (truncated at 70 chars; full text in tooltip) · Country · Assigned To · Ext. ID · Order Numbers · Critical for Sign-Off (inline dropdown) · Next Step (click to edit) Filters Area (multi) Type (multi) Status (multi) Assigned To (multi) Critical for sign-off (multi) Show passed & dropped toggle Hidden by default Statuses: Passed · Passed pending Solman · Dropped (configurable in settings.yaml) Inline edits (AJAX) Critical for sign-off — inline <select> in the table; saves on change → POST /spillover/<id>/critical
Next step — click cell to open the Details popup (see below) Details button Opens the Details popup for the row. Editable fields: Report Group · Comment for Sign-Off · Importance for Sign-Off · Next Step. Saves via POST /spillover/<id>/annotation (AJAX, no page reload). Clicking Next Step cell in the table opens the same popup. Order details button Opens a popup table of order-number lines for this spillover item. Columns: Order Number · Comment. Each field is a plain <input> — editable directly, saves on blur via POST /order-details/<detail_id>/update. + Add row creates a new DB record immediately then focuses the first input. ✕ deletes the row via POST /order-details/<detail_id>/delete. Uses the generic order_details table (entity_type='spillover', entity_id=spillover_id). Comments button Opens the Comment History popup: shows the imported comment field read-only as a reference, plus an editable textarea for the authored comment history. Saves via POST /spillover/<id>/comment. Notes button Links to /spillover/<id> (Spillover Detail). Shows note count badge when notes exist (subquery on notes table, computed per row). Order details API GET /order-details/<entity_type>/<entity_id> — list rows as JSON
POST /order-details/<entity_type>/<entity_id>/add — create empty row, returns {ok, id}
POST /order-details/<detail_id>/update — save order_number + comment
POST /order-details/<detail_id>/delete — remove row
Generic design — works for any entity type, not just spillover. Table source spillover LEFT JOIN spillover_annotations + note_count subquery on notes

Tracks agenda topics per named meeting. Topics are added inline via a quick-add form at the top; status, overall topic, and topic text all edit in-row via AJAX. No detail page — everything is managed on this one screen.

Columns Meeting · Overall Topic · Topic · Status · Inline note · Notes count Filters Meeting (dropdown) Status (dropdown, default: planned) Meeting options Balazs · GPO · Sync&Solve · DTC O2C Daily · Sales ECOM daily · Other Status values planned · discussed · future — default filter shows planned only Quick-add form Meeting selector + topic text + optional Overall Topic → POST /meeting-prep/add Overall Topic Optional grouping label: CS Retail · CS ECOM · CS General · ROE Retail · ROE ECOM · ROE General · Orga · AI · Other. Inline <select> per row; saves on change → POST /meeting-prep/<id>/overall_topic. Topic inline editing Hover the topic cell to reveal a pencil (✎) button. Click to edit in place; save with Enter or Save button → POST /meeting-prep/<id>/topic. Esc cancels. Source entity badge When an item was added from a Defect or Retail detail page, the Topic column shows a coloured badge below the topic text: purple for defects (defect ID + SolMan name), green for retail test cases (test case ID / country / scenario). Each badge is a link back to the source detail page. Source entity badge Items added from a Defect detail page show a purple badge (defect ID + SolMan name → link to defect). Items added from a Retail detail page show a green badge (test case ID / country / scenario → link to test case). Manual items show no badge. Export actions Export agenda → GET /meeting-prep/agenda opens in new tab: styled HTML report with coloured section banners per overall topic. Includes Download HTML and Print buttons.
Copy to clipboard → plain text grouped by overall topic order with section labels.
