Core
Dashboard GET /

The home screen. Shows a grid of action cards — one per major module. Acts as the main navigation hub since there is no persistent sidebar.

Cards Inbox (pending count badge) · Shelf (item count badge, purple accent) · Import · Defects · Core South Spillover · ECOM Gatekeeper · Retail · Meeting Prep · To-Do · Follow-ups · Links · Contacts · CS Follow-Up Tracker · Encouragements (undelivered count badge) · Enhancements (open count badge) Primary action Run Import — triggers a POST to /import and redirects to the Import Result screen Navigation Each card contains a link button to the corresponding list screen. The site title in the header also links back here from every page.
Inbox GET /inbox

Daily capture pad. Paste notes and screenshots during the day, then file each item to its destination (Defect, Retail test case, Spillover, Test Learning, Follow-up, or Shelf) when ready. Unfiled items stay here until moved or deleted. Dashboard card shows pending count badge.

Add item Heading (optional) + large textarea + 📷 Add screenshot or 📎 Attach file or Ctrl+V paste. Items appear in reverse-chronological order. Per-item actions Edit — inline toggle; updates heading and note text
Delete — removes item and all attachments
File › — opens filing picker: choose type, search by name, select target, then Move › Filing behaviour For Defect / Retail / Spillover / Test Learning / Follow-up: single SQL UPDATE — the note row and all its attachments move to the target entity. Filed items appear immediately in the target's detail page notes log.
For Shelf: a new Shelf item is created from the inbox heading; area and category are entered in the picker (both optional); the note is then re-parented to the new shelf item. Route: POST /inbox/<note_id>/file-to-shelf. Attachment support Images (PNG/JPG/GIF/WEBP) shown as thumbnails. Documents (PDF/DOC/DOCX/XLS/XLSX/PPT/PPTX/TXT/CSV/MSG/EML/ZIP) shown as download links. Ctrl+V pastes a Snipping Tool image into the last-hovered item. ✕ to delete any attachment. Table source notes (entity_type='input', entity_id='inbox') + attachments
Import Result POST /import

Shown after running an import. Displays a per-tab summary of what happened — how many rows were inserted, updated, or skipped, plus archive status and any skip-log file path.

Per-tab summary Defects · Core South Spillover · Retail (only tabs with enabled: true in settings.yaml appear) Counts shown Inserted · Updated · Skipped (blank key) · Skipped (duplicate) · Ignored (blank rows) Archive info Archive status (archived / skipped_duplicate / disabled) and path of archived file Skip log Path to the per-run CSV if any rows were skipped Navigation Back to Dashboard
Shelf
Shelf — List GET /shelf

Catch-all store for notes and observations that don't belong to a specific defect, test case, or spillover item yet. Items have a heading, optional area, and optional category. Dashboard card shows total item count with a purple accent. Reachable from the Inbox filing picker (Shelf option) or directly from the dashboard.

Quick-add form Heading (optional) · Area (optional) · Category (optional) → POST /shelf/add Filters Area (multi-select dialog) Category (multi-select dialog) — same <dialog>-based multi-select pattern as Spillover and CS Follow-ups Columns Heading (link → detail) · Area (blue pill) · Category (purple pill) · Added date · Notes count badge Combine feature Each row has a checkbox. When 2+ rows are checked a "Combine N selected ›" button appears above the table. Clicking it opens a modal listing the selected items as radio buttons — pick the primary item to keep; all notes from the others are moved into it and the secondary rows are deleted. Route: POST /shelf/combine (fields: primary_id + item_ids[]). Redirects to the primary item's detail page. A "Select all" checkbox in the header selects every visible row. From Inbox Selecting Shelf in the Inbox filing picker shows area + category text inputs (no search — a new shelf item is always created). Route: POST /inbox/<note_id>/file-to-shelf. The inbox item's heading becomes the shelf item's heading; the note is re-parented. Flash banners Item added · Item deleted Table source shelf LEFT JOIN notes (note_count subquery) Routes GET /shelf — list (accepts area[] + category[] query params)
POST /shelf/add — create item
POST /shelf/combine — merge selected items into primary
Shelf — Detail GET /shelf/<id>

Full view of a single shelf item. Metadata (heading, area, category) is editable inline. Below the metadata card: a complete notes module with add, edit, delete, file attachments, and Ctrl+V paste — identical to Defect and Retail detail pages.

Editable metadata Heading · Area · Category — all in one inline form row; Save → POST /shelf/<id>/update → redirect with ?saved=1 flash Notes section Full notes log: heading + body + timestamp. Per-note Edit and Delete. 📷 Add screenshot and 📎 Attach file per note; images as thumbnails, documents as download links. Ctrl+V pastes a clipboard image (e.g. from Snipping Tool) into the last-hovered note. Delete item Delete button in page header — confirm dialog → POST /shelf/<id>/delete → cascades to notes + attachments (disk files also removed); redirects to shelf list. Flash banners Saved · Note added · Note saved · Note deleted Navigation Breadcrumb (Shelf › Item heading) · ← Back to Shelf Note routes GET/POST /shelf/<id>/notes/add
GET/POST /shelf/<id>/notes/<note_id>/edit
GET/POST /shelf/<id>/notes/<note_id>/delete Table source shelf + notes (entity_type='shelf') + attachments

Defects — List GET /defects

Paginated list of all imported defects with inline filters. The default view hides statuses "Withdrawn" and "Confirmed" (configurable in settings.yaml).

Columns Defect ID · SolMan Name · Blocked TCs · Channel · Status · Priority · Assigned To · Date Reported · Prod · DTC O2C (inline checkbox) · Daily (inline checkbox) Blocked TCs column Count of Retail test cases whose defect_id_ref references this defect. Computed as a subquery (LIKE match — not stored). Shows — when zero. When non-zero, displayed as a red badge link that opens the Retail list pre-filtered by that defect ID (?search_defect=<id>). Filters Search (defect ID) Channel Status (multi-select) Action needed DTC O2C Daily Show all toggle Row actions Click row → Defect Detail · Add note shortcut per row · Blocked TCs badge → Retail list filtered by defect ID Flash banners Green "Note added." banner when returning from a successful note add Table source defects LEFT JOIN defect_annotations + subquery on retail.defect_id_ref

Full view of a single defect — imported fields on the left, editable annotation form on the right, and a notes log at the bottom.

Read-only fields All columns from the defects table (channel, status, priority, dates, scenario, etc.) Editable annotation Description · Business Impact · Reach · Retest Needs · Next Step · Action Needed (checkbox) · Comments · DTC O2C (checkbox) · DTC O2C Responsible (text) · To discuss on daily (checkbox)
Save → POST to same URL → redirect with ?saved=1 flash
DTC O2C and Daily can also be toggled inline from the list view (AJAX, no reload) Notes section List of all notes for this defect (heading, body, timestamp). Per-note Edit and Delete links. Each note shows an attachment area with 📷 Add screenshot and 📎 Attach file buttons. Images shown as thumbnails; documents as download links. Hover an attachment to reveal the ✕ remove button. Ctrl+V pastes a clipboard image (e.g. from Snipping Tool) into the last-hovered note. Add to Meeting Prep 📅 Add to Meeting Prep button at the bottom — expands an inline form: pick meeting + edit pre-filled topic (defect ID + SolMan name). POSTs to /meeting-prep/add with source_entity_type=defect and source_entity_id. Redirects back with green confirmation banner. Flash banners Annotation saved · Note added · Note saved · Note deleted · Added to Meeting Prep Navigation ← Back to Defects · Add note · 📅 Add to Meeting Prep Table source defects LEFT JOIN defect_annotations + notes + attachments
Defects — Note Add / Edit / Delete GET /defects/<id>/notes/add  ·  /notes/<id>/edit  ·  /notes/<id>/delete

Three separate screens for managing notes on a defect. Add and Edit share the same note_form.html template (distinguished by a mode variable). Delete is a confirmation page. Screenshot attachments are managed via AJAX on the detail page itself (no separate screen).

Add / Edit fields Heading (optional) · Note text (required — shows inline error if empty) Delete Confirmation page showing the note heading/body. Requires a second POST to confirm deletion. Return flow Add can return to either the defect detail or the defects list (controlled by return_to query param). Edit and Delete always return to the defect detail. File attachments Upload images via 📷 Add screenshot or Ctrl+V paste; upload documents via 📎 Attach file (AJAX, no page reload). Images shown as thumbnails; documents as download links. Delete via ✕ on hover. All files stored in data/uploads/. Routes: POST /notes/<note_id>/attachments/add · POST /notes/<note_id>/attachments/<id>/delete · GET /uploads/<filename>. Tables written notes (entity_type = 'defect') · attachments (on upload/delete)
Core South Spillover
Spillover — List GET /spillover

Horizontally-scrollable frozen-pane table of Core South Spillover items. The row number (#) and Name columns are frozen (sticky); all others scroll. Rows are compact — Next Step and Order Numbers are clamped to 3 lines with tooltips showing the full text. Long annotation fields (Importance, Comment for Sign-Off, Report Group) live in the Details popup only. Four action buttons per row open dialogs or navigate to detail.

Column order Frozen: # (Excel row) · Name (220 px, full text in tooltip)
Scrollable: Buttons → Status → Next Step (≤3 lines, clamp-3) → Area → Order Numbers (≤3 lines, clamp-3, tooltip) → Country → Assigned To → Ext. ID → Critical for Sign-Off Filters Area (multi) Type (multi) Status (multi) Assigned To (multi) Critical for sign-off (multi) Show passed & dropped toggle Hidden by default Statuses: Passed · Passed pending Solman · Dropped (configurable in settings.yaml) Inline edits (AJAX) Critical for sign-off — inline <select>; saves on change → POST /spillover/<id>/critical
Next step — click cell opens the Details popup Status Report button In page header — navigates to the Spillover Status Report selection screen (/spillover/report). Details button Opens the Details popup. Editable: Next Step · Importance for Sign-Off · Comment for Sign-Off · Report Group. Saves via POST /spillover/<id>/annotation (AJAX). Clicking the Next Step cell also opens this popup. Order details button Opens a popup table for this item's order-number lines. Columns: Type · Order Number · Comment · S4 docs (checkbox — green when ticked). Each text field saves on blur; checkbox saves immediately on change. Both call POST /order-details/<detail_id>/update.
When any order line has S4 docs checked, the button shows a green ✓ badge in the list so the status is visible without opening the popup. Badge updates live as the checkbox is toggled.
Uses the generic order_details table (entity_type='spillover'). Comments button Opens the Comment History popup: shows imported comment read-only, plus editable comment history textarea. Saves via POST /spillover/<id>/comment. Notes button Links to /spillover/<id> (Spillover Detail). Shows note count badge when notes exist. Order details API GET /order-details/<entity_type>/<entity_id> — list rows as JSON (includes docs_in_s4)
POST /order-details/<entity_type>/<entity_id>/add — create empty row → {ok, id}
POST /order-details/<detail_id>/update — save order_type, order_number, comment, docs_in_s4
POST /order-details/<detail_id>/delete — remove row
Generic — works for any entity type. Table source spillover LEFT JOIN spillover_annotations + note_count subquery on notes. S4 docs badge pre-computed server-side from order_details. Report Comments section Below the main table: inline-editable list of free-text bullets that will appear in the "Additional" section of the Spillover Status Report. + Add bullet creates an empty row immediately (AJAX, no page reload); each row saves on blur; ✕ deletes. APIs: POST /report-comments/spillover/add · POST /report-comments/<id>/update · POST /report-comments/<id>/delete.

Spillover — Status Report (view / print) GET /spillover/report/view

Standalone print-ready report titled Spillover Status Report — from DTC Perspective. Professional document layout — dark header bar per item, metadata strip (Area · Status · Critical for sign-off), next step with left-border accent, and an order details table when entries exist. Items are sorted: Critical → Slightly critical → Not critical → unset. Summary stats in header. Opens in a new tab.

Per-item sections Numbered header (dark navy) · Metadata strip (3 cells: Area · Status · Critical) · Next Step (pre-wrap, left-border accent) · Order Details table (Type · Order Number · Comment · S4 docs ✓) Sort order Items are always sorted critical-first: Yes → Slightly → No → unset. Selection order in the source table is overridden. Critical colour coding Yes → red · Slightly → amber · No → green · — → muted S4 docs in report Green ✓ shown in the S4 docs column of the order details table for any line where the flag is set; — (muted) otherwise. Additional section Below all item cards: bullet list of report comments (from report_comments where report='spillover'). Only shown when comments exist. Edit the bullets on the Spillover list page. Included in the Downloaded HTML snapshot. Passed items Items with status Passed or Passed, pending solman are visually celebrated: card header turns green and a 🎉 icon appears before the name. A "Closed this round" summary section appears at the bottom of the report (above Additional), listing each passed item with name · area · exact status. Both the green header and summary background are forced via print-color-adjust: exact so they survive PDF export. Actions (screen only) Print — browser print dialog · Download PDF — server-generated A4 PDF via WeasyPrint (GET /spillover/report/pdf) · Download HTML — saves a self-contained snapshot (includes Additional section) · 📋 Copy for Teams — copies Teams-formatted markdown (name, next step, orders, Additional bullets) to clipboard · ← Edit selection Teams copy format Header line with title + date, then per item: bold name, next step (multi-line preserved), order bullets ([Type] OrderNo — Comment ✓ S4). If Additional bullets exist, a bold "Additional" line followed by bullet lines is appended. Toast confirms copy. Print / PDF optimised Toolbar hidden in print; item cards have break-inside: avoid; background colours forced via print-color-adjust. PDF uses A4 with 2 cm margins (@page rule). Data source spillover_report_selection JOIN spillover JOIN spillover_annotations; order details loaded per item from order_details (entity_type='spillover').
