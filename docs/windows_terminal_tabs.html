🗂 Windows Terminal — Claude Code project tabs

How the "new tab" dropdown gets a new project profile that opens Claude Code directly in a project folder. Written 2026-07-12; at that point the dropdown held: RAG, Planning Dashboard, UAT, Chaosco, Amazon Ledger, Icebreaker, Kursmachen, Slack Download, WalkThrough Course.

Where the dropdown lives

One file controls everything

C:\Users\AI_Agents\AppData\Local\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json

  • Every entry in profiles → list is one row in the dropdown.
  • Windows Terminal picks the change up immediately when the file is saved — no restart needed.
  • Shortcut without hunting for the path: Windows Terminal → dropdown arrow → Settings (Ctrl+,) → Open JSON file (bottom-left).
  • Before editing by hand, copy the file to settings.json.backup-YYYY-MM-DD next to it. If the JSON is broken, Terminal shows an error banner and keeps running on the last good version — restore the backup and save again.

Anatomy of a project profile

Copy an existing Claude Code block (Chaosco is a good template), change four things: guid (must be new and unique), name, startingDirectory, tabTitle + tabColor.

{
    "guid": "{NEW-GUID-HERE}",
    "name": "My Project",                // text shown in the dropdown
    "commandline": "%SystemRoot%\\System32\\WindowsPowerShell\\v1.0\\powershell.exe -NoExit -Command claude",
    "startingDirectory": "C:\\AIProjectsSandbox\\ClaudeProjekte\\MyProject",
    "tabTitle": "MyProj",                // short text on the tab itself
    "tabColor": "#0e7490",               // pick one not used yet
    "suppressApplicationTitle": true,    // keeps the tabTitle fixed
    "hidden": false
}
  • Get a fresh GUID in PowerShell: [guid]::NewGuid() — then wrap it in { } in the file. (JSON comments above are illustration only — Windows Terminal tolerates them, but keep the real file clean.)
  • Backslashes in paths are doubled (\\) — JSON escaping.
  • The block goes anywhere inside profiles → list; dropdown order = list order, so keep the Claude Code projects together above cmd/Azure.

The lazy way — prompt for a Claude Code instance

Copy-paste this into any Claude Code session
Please add a new project profile to my Windows Terminal dropdown.

The settings file is:
C:\Users\AI_Agents\AppData\Local\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json

Steps:
1. Back the file up first (settings.json.backup-<today> next to it).
2. In profiles.list, copy the pattern of the existing "Chaosco" profile:
   PowerShell running claude via
   "%SystemRoot%\\System32\\WindowsPowerShell\\v1.0\\powershell.exe -NoExit -Command claude",
   suppressApplicationTitle true, hidden false.
3. Generate a fresh GUID for the new entry.
4. startingDirectory = the project folder I name below.
5. Give it a short tabTitle and a tabColor that is NOT already used by
   another profile in the file (list the colors in use if unsure).
6. Insert it after the last Claude Code project profile (before the
   cmd.exe entry) and verify the file still parses as valid JSON.
7. Also update the color table and profile list in
   C:\AIProjectsSandbox\ClaudeProjekte\chaosco\docs\windows_terminal_tabs.html
   so the documentation stays current.

Project folder: <PASTE FULL PATH HERE>
Dropdown name:  <NAME HERE>
Good to know
  • Remove a project: delete its block from profiles.list (or set "hidden": true to keep it but hide it from the dropdown).
  • Rename / recolor: edit name / tabTitle / tabColor in place — takes effect on the next new tab.
  • Backups on disk: settings.json.backup-2026-07-05 and settings.json.backup-2026-07-12 already exist next to the settings file.
  • The defaultProfile at the top of the file decides what a plain "new tab" (the + button) opens — currently plain Windows PowerShell, not a project.
