Dashboard at a glance

One mini-card per dashboard card, in the fixed card order (daily work first). Full per-screen detail: docs/screens.html · tables: docs/database_schema.html · big picture: docs/architecture.html. Updated 2026-07-18.

Inbox count

The daily capture pad. Paste notes and screenshots as they happen, tag them with Order/SolMan/Jira IDs or a route-to, then file each item to its destination — by hand or via ⚡ Auto-file.

/inbox

Import DTC ROE Tracking

One click: finds the newest Excel export, archives it, and upserts all enabled tabs (Defects · Spillover · Retail · ECOM) into the database. Idempotent — safe to run any time.

POST /import

Core South Spillover

The spillover items with your annotations (importance, next steps, comment history, sign-off fields). Opens on the "In report" view; feeds the Spillover Status Report.

/spillover

Retail

All Retail test-case rows with filters, inline next steps, the Comments dialog, and the ⚠ data check. Source for the Retail Status Report and the Requirements Tracker.

/retail

ECOM Gatekeeper

Sales-facing pre-handoff check on the Jira tickets: Active gatekeeping / Back with Sales sections, next steps, orders, reporter filter, and the 📄 Sales report (also per reporter).

/ecom-gatekeeper

ECOM

ECOM test cases from the Excel, joined live with the shared Jira store (status, assignee, comments). Comments dialog, reporter filter, ⚠ data checks, own status report.

/ecom/

Defects

All imported defects with search, channel filter (ECOM/RETAIL, casing-proof), status filters, blocked-test-case counts, and the annotation flags (action needed, DTC O2C, daily).

/defects

Known Prod Defects count

Hand-maintained register of defects known to exist in production — outlives the spillover work; feeds the known-issues section of the sign-off report.

/prod_defects

Export & Backup

Two jobs: dated HTML + PowerPoint snapshots of both status reports to report_export/ (+ 📧 Email Reports), and the 💾 DB backup — database + screenshots to the external drive, with a last-backup reminder.

POST /export-reports · POST /backup

Meeting Prep

Topics to raise per meeting — filter by meeting, tick items off as discussed.

/meeting-prep

Retail Requirements Tracker unresolved

Tracks that every Retail requirement is tested across countries — derived LIVE from Retail statuses, never stored as yes-marks. Board + payment-methods matrix + import & admin.

/retail-tracker/board

Topics active

What you're actively working on: per topic a big formatted workpad (first!), next steps, links, connected defects/test cases/spillover items, and notes.

/topics/

Follow-ups

Per-person follow-up items with next-contact dates and notes.

/followups

Links

Useful-links store — filter by area/tool/tag, one-click copy. Inbox items can be routed here.

/links

Contacts

Contact directory (name, email, area, topic, tags) with notes per contact. Inbox items can be routed here.

/contacts

Shelf count

Catch-all archive for notes that don't belong to a specific entity yet — the counterpart to Topics (Shelf = parked, Topic = active).

/shelf

Encouragements undelivered

Positive observations about people, with a delivered flag — so appreciation actually gets said.

/encouragements

CS Follow-Up Tracker

Core South follow-up items by area and Jira ID, with next steps, owner, and notes.

/cs_followups

To-Do

Plain task list with priority, due date, owner, and notes.

/todos

Teams Chats

Registry of Teams chats & channels — pin the important ones for the floating 💬 widget, attach any to Retail / ECOM / Gatekeeper / Spillover items.

/teams-chats/

Message Types

Reference for the ✉️ issue-message builder: message types with their TIBCO / IIB APIs and comments.

/message-types

Enhancements open

Tracked improvement ideas for chaosco itself — captured from every page via the floating panel.

/enhancements/page
On every page, not on a card: the floating 🔍 order-number search · the floating 💬 pinned-chats widget · the ➕ Enhancements capture panel.
